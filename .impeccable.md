# SnapCos Design Context

## Design Context

### Users
- **Vendors:** Independent costume owners and small businesses. They use the platform to manage listings, subscriptions, and reservations. They need a high-utility, reliable "SaaS" interface that minimizes friction.
- **Renters:** Enthusiasts, cosplayers, and event-goers. They browse for inspiration and book rentals. They need an immersive, high-end "Editorial" experience that highlights visual content.

### Brand Personality
**Modern, Efficient, Trustworthy.**
SnapCos balances the functional robustness of a modern SaaS tool with the visual elegance of a premium editorial marketplace. It should feel "expert" and "intentional."

### Aesthetic Direction
- **Visual Tone:** Refined utilitarianism. High-performance UI meeting magazine-style layout.
- **References:** **Airbnb** (clarity, discovery, trust), **SSENSE** (editorial minimalism, bold type).
- **Theme:** **System/Both** (Adaptive light/dark support).
- **Key Elements:** Crisp 1px borders, warm monochrome palette, generous whitespace, zero heavy shadows.
- **Typography:** *Newsreader* (Serif display) for hierarchy and character; *DM Sans* (Sans-serif) for high-legibility UI and body text.

### Design Principles
1. **Utility-Driven Elegance:** Every UI element must serve a functional purpose. Aesthetic beauty comes from proportion and typography, not decoration.
2. **Content-Forward:** The interface recedes to let costume photography take center stage.
3. **Seamless Multi-Platform:** Visual consistency is absolute between the Next.js web platform and the Expo mobile app.
4. **SaaS-Grade Precision:** Interactions must be responsive and stateful. Clear feedback for vendor workflows (approvals, listing management) is critical.
