## Design Context

### Users
- Boutique costume owners and small rental vendors preparing curated collections for the marketplace
- They use the vendor area to apply, build private drafts, publish polished listings, and review renter activity
- They need a workspace that feels trustworthy, premium, and operationally clear — with theatrical brand energy

### Brand Personality
- Theatrical
- Playful
- Premium
- Trustworthy

### Aesthetic Direction
- **North Star:** "Snap Into Character" — coral, gold, and curtain black from the logo
- **Storefront:** Utility-forward marketplace layout (pill search top bar, filter sidebar, results toolbar, filter chips, dense card grid) — see `components/marketplace/`
- **Vendor dashboard:** Rounded-xl card surfaces on `page-bg` / `bg-muted/40` backgrounds; coral-filled pill active nav in sidebar
- **Inventory:** Reuses marketplace `ResultsToolbar`, filter chips, and dense `CostumeCard` grid for listings
- Fraunces display typography, DM Sans UI body, soft coral-tinted hover shadows, snap button micro-interactions
- Explicitly avoid colorless editorial minimalism, flat admin-card grids, tall hero-only browse, and generic SaaS filler layouts

### Design Principles
- Make vendor state visible: application, draft, live, and moderated states should feel distinct and understandable
- Let trust lead: identity review, approval gating, and listing visibility rules should read clearly without sounding punitive
- Keep the collection central: listings should feel like marketplace products with scannable price and metadata — not anonymous inventory rows
- Reward momentum: even during review, the interface should invite meaningful next steps instead of dead ends
- Use coral for primary CTAs everywhere — never monochrome foreground-filled buttons
- Show real data on listing cards only — no fabricated ratings, sold counts, or discount prices
